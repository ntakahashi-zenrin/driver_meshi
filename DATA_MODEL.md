# データ構造定義：トラックドライバー飯 SNS（サンプル）

このファイルは、サンプルSNSで使うデータの設計図です。
Appwriteのデータベース（コレクション）とストレージは、この定義に沿って作ってください。

この設計は「Appwrite専用」ではありません。SNSとして必要なデータの持ち方を
標準的に書いたものなので、将来Supabase等の別の土台に移しても通用します。
（＝開発者に引き継ぐ際、技術が変わっても「作るものの中身」はこの定義がそのまま使えます）

---

## 全体像

サンプルでは、次の6種類のデータ（Appwriteでは「コレクション」）を持ちます。

1. drivers（ドライバー＝ユーザーのプロフィール）
2. meal_posts（飯投稿）
3. gallery_posts（ドライバーギャラリー投稿）
4. follows（フォロー関係）
5. likes（いいね）
6. places（場所マスター）※飯投稿の「場所」を地図で指定したとき、ここに1件ずつ蓄積される

加えて、写真を保存する「ストレージ」を1つ用意します（飯写真・ギャラリー写真・アイコン等）。

認証（ログイン）はAppwriteの認証機能をそのまま使い、1ユーザー＝1ドライバーとします。

---

## 1. drivers（ドライバープロフィール）

企画書セクション15・20に対応。1人のドライバーの「名刺ページ」の中身。

| 項目 | 型 | 説明 | 例 |
|---|---|---|---|
| user_id | 文字列 | Appwrite認証のユーザーIDと紐づく | （認証ID） |
| nickname | 文字列 | ニックネーム | 関西便のタカ |
| avatar_id | 文字列 | アイコン画像（ストレージのファイルID） | |
| cover_id | 文字列 | メインビジュアル画像（同上） | |
| body_type | 文字列(選択式) | 車種・ボディタイプ | 大型ウイング |
| run_style | 文字列(選択式) | 運行スタイル | 長距離 |
| area | 文字列(選択式) | 主な走行エリア | 関東〜関西 |
| years | 数値 | ドライバー歴（年） | 12 |
| bio | 文字列 | 一言プロフィール | 大型車OKの食堂と深夜ラーメン中心 |
| fav_genres | 文字列の配列(タグ) | 好きな飯ジャンル | [デカ盛り, ラーメン, 定食] |
| fav_places | 文字列の配列(タグ) | よく投稿する場所 | [SA・PA, 幹線道路沿い] |
| priorities | 文字列の配列(タグ) | 重視するポイント | [大型車OK, 深夜営業] |
| badges | 文字列の配列 | 獲得バッジ | [デカ盛りハンター, SA・PAマスター] |

※入力は選択式・タグ式中心（企画書セクション14）。body_type/run_style/area は
　あらかじめ用意した選択肢から選ぶ形にする。

## 2. meal_posts（飯投稿）

企画書セクション9.1・19・22に対応。サービスの中心となる投稿。

| 項目 | 型 | 説明 | 例 |
|---|---|---|---|
| driver_id | 文字列 | 投稿者（driversのuser_id） | |
| image_id | 文字列 | 飯写真（ストレージのファイルID） | |
| dish | 文字列 | メニュー名 | デカ盛り唐揚げ定食 |
| shop | 文字列 | 店名 | 海老名SA 下り |
| price | 数値 | 金額（円） | 1280 |
| location | 文字列 | 場所（表示用テキスト） | 神奈川県海老名市大谷南 |
| place_id | 文字列 | 場所マスター（placesのID）への参照。地図で選んだ場合に入る。空でもOK | |
| lat | 数値(小数) | 緯度（地図で選んだ位置） | 35.4344 |
| lng | 数値(小数) | 経度（地図で選んだ位置） | 139.3899 |
| oogata_ok | 真偽値 | 大型車OKか | true |
| amenities | 文字列の配列(タグ) | 店の属性 | [大型車OK, 駐車場広め, 深夜営業] |
| genres | 文字列の配列(タグ) | 飯ジャンル | [長距離, デカ盛り, 深夜飯] |
| comment | 文字列 | ひとこと | 深夜でも助かる一杯 |
| created_at | 日時 | 投稿日時 | |

※`place_id`・`lat`・`lng` は今回の地図投稿で追加。地図で場所を選ぶと places に1件登録され、
　その ID を `place_id` に保存する。緯度経度は投稿自体にも控え（snapshot）として持たせ、
　SA・PAの自販機飯や路肩弁当など「マスターに載らない場所」でも位置だけは残せるようにする。

## 6. places（場所マスター）

飯投稿で「地図で場所を選ぶ」と、選んだ場所がここに1件ずつ蓄積される“店舗・スポットの台帳”。
将来の「店舗探し（大型車OK・深夜営業などで探す）」機能の土台になる。ユーザー投稿で増えていくが、
重複（同じ店が表記違いで二重登録）の整理は、当面は管理者がAppwriteコンソールで行う。

| 項目 | 型 | 説明 | 例 |
|---|---|---|---|
| name | 文字列(必須) | 場所名・店名 | 札幌トラックステーション |
| address | 文字列(必須) | 住所 | 北海道札幌市厚別区厚別東5条1-1-2 |
| pref_code | 数値(必須) | 都道府県コード（北海道=1 〜 沖縄=47） | 1 |
| lat | 数値(小数, 必須) | 緯度 | 43.0319 |
| lng | 数値(小数, 必須) | 経度 | 141.4940 |
| category | 文字列(選択式, 必須) | 場所カテゴリ | 道の駅 / トラックステーション / SA・PA / 店舗 |
| hours_text | 文字列 | 営業時間（自由テキスト） | 9:00〜17:00（土日祝休） |
| phone | 文字列 | 電話番号 | 011-897-9101 |
| midnight_open | 真偽値 | 深夜営業あり | false |
| early_open | 真偽値 | 早朝営業あり | false |
| oogata_ok | 真偽値 | 大型車OK | true |
| open_24h | 真偽値 | 24時間対応 | false |
| truck_parking | 文字列 | 大型車の駐車台数目安（自由記述。停められる規模感を言葉で書ける） | 大型車なら2台程度 |
| shower | 真偽値 | シャワーあり | false |
| wifi | 真偽値 | WiFiあり | false |
| created_by | 文字列(必須) | 登録者ID（ユーザーのuser_id、または admin） | admin |
| created_at | 日時(必須) | 登録日時 | |
| updated_at | 日時(必須) | 更新日時 | |

※添付CSV（DB項目.csv）の項目をベースに採用。判断が必要だった2点は次の通り：
- **大型車駐車台数目安**：**文字列（自由記述）**として定義。「大型車なら2台程度」のように規模感を言葉で書けるようにする
  （数字限定だと表現しきれないため）。
- **場所ID**：マスターIDにはAppwriteが自動で振る一意ID（`$id`）をそのまま使う。飯投稿の `place_id` もこの `$id` を参照する。
  当初CSVにあった `SPOT-0001` 形式の管理コードは、`$id` があれば運用上不要なため採用しない。

## ストレージ（画像保存）

- バケットを1つ作る（例：media）。飯写真・ギャラリー写真・アイコン・カバー画像を保存。
- 画像はアップロード後にファイルIDが返るので、それを各コレクションの *_id 項目に保存する。

---

## フィード（タイムライン）の作り方の方針

- **飯タイムライン（未フォロー含む全員が見る入口）**：meal_posts を新しい順に表示。
  各投稿に、driver_id から引いたドライバー情報（ニックネーム・車種・エリア・タグ）を必ず添える。
- **フォロー中タイムライン**：ログイン中ユーザーの follows から followee_id 一覧を取得し、
  その人たちの meal_posts ＋ gallery_posts ＋ バッジ獲得を時系列で混在表示する（企画書セクション10・21）。
  ※バッジ獲得は今回はサンプルとして簡易表現でよい（drivers.badges の更新を活動として見せる等）。

## サンプルデータ（初期投入）

検証しやすいよう、プロトタイプ（driver_meshi_prototype.html）に登場する3人
（関西便のタカ／地場めしユウ／夜景トレーラー）と、その飯投稿・ギャラリー投稿を
ダミーデータとして投入できるようにしておくと、画面の見栄えが確認しやすい。
写真はフリー素材または仮画像でよい（本物のトラック飯写真である必要はない）。
