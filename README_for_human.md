# 【あなた向け】Appwriteでサンプルを手元に構築する手順

このフォルダ一式を Claude Code に渡して、トラックドライバー飯SNSの
「動くサンプル」を、あなたのPCの中（ローカル）に作ってもらうための手順書です。

このフォルダの中身：
- `CLAUDE.md` … Claude Codeが最初に読む指示書（あなたは編集しなくてOK）
- `DATA_MODEL.md` … サンプルで使うデータ構造の設計図（編集不要）
- `truck_driver_meshi_kikakusho.md` … 企画書
- `driver_meshi_prototype.html` … 動くプロトタイプ（見本）

---

## このサンプルで何が分かるのか（先に目的を共有）

手元で動くサンプルを作る目的は、企画書セクション26の「3つの検証」を、
絵ではなく“実際に触って”確かめることです。
- 飯投稿は見て楽しいか
- 投稿者（ドライバー）に興味を持つか
- ドライバーが投稿したくなるか

このサンプルは「Appwriteという裏側の仕組み＋プロトタイプの画面」を
つないで、ログイン・投稿・フォローなどが実際に動く状態にしたものです。

> 補足：Appwriteは「SNSの完成品」ではなく「SNSを作るための裏側の土台」です。
> なので開発者に話すときは「Appwriteで」ではなく、「この企画書とデータ構造のSNSを、
> Appwriteを土台に作る」と伝えると正確に通じます（CLAUDE.md にもその旨を記載済み）。

---

## 前提（必要なもの）

- **Docker Desktop が起動していること**（導入済みとのことなので、起動だけ確認）。
  Appwriteは Docker の上で動きます。最低2GB、できれば4GB以上の空きメモリが望ましいです
  （出典：Appwrite公式 / selfhosting.sh, 2026年。Appwriteは複数コンテナで動くため）。
- **Claude Code がインストール済みであること**（未導入なら、別途お渡しした
  README_for_human.md のステップ1を参照してください）。
- すでにお持ちの GitHub アカウント（必須ではないが、コード管理に使えます）。

---

## 手順

### ステップ1：このフォルダをPCの分かりやすい場所に置く
デスクトップなど、分かりやすい場所にフォルダごと置いてください。

### ステップ2：Docker Desktop を起動しておく
Docker Desktop アプリを起動し、動いている状態にしておきます。

### ステップ3：そのフォルダの場所で Claude Code を起動する
ターミナルで、置いたフォルダに移動して `claude` と打ちます。例：
```
cd ~/Desktop/driver-meshi-appwrite
claude
```

### ステップ4：最初の声かけ
Claude Code が起動したら、次のように打つだけです。
```
CLAUDE.md、DATA_MODEL.md、truck_driver_meshi_kikakusho.md を読んでから、
Appwriteをローカルに立ち上げて、サンプルSNSを段階的に作ってください。
まず段階1（Appwrite起動）から始めて、各段階の終わりに動作確認の方法を教えてください。
```

あとは Claude Code が、Appwriteの起動 → データ構造の作成 → 画面の接続、と
段階的に進めてくれます。各段階の終わりに「ブラウザでこのURLを開くと何が見える」と
教えてくれるので、そこで実際に触って確認してください。

## 進めるうえでのコツ・注意

- **Appwriteの起動には数分かかることがあります**（Mac/Windowsでは特に。正常な挙動です。
  出典：Appwrite公式, 2026年）。「固まった?」と思っても少し待ってください。
- **起動が終わると、ブラウザで http://localhost を開くとAppwriteの管理画面が出ます。**
  ここで管理者アカウントを作る場面があります。これはあなたのPCの中だけの話なので安全です。
- **「データを全部消す」系のコマンドだけは要注意。** CLAUDE.md で「破壊的な操作は必ず確認」と
  指示してありますが、`docker compose down -v` のような全削除コマンドを求められたら、
  本当に消してよいか一度立ち止まってください。
- **エラーが出たら、そのまま Claude Code に貼って「これは何?どうする?」と聞けばOK。**
- **意味の分からないコマンドは「これは何のため?」と遠慮なく聞いてください。**

## このサンプルの“その後”について（今は読むだけでOK）

- このサンプルはあなたのPCの中だけで動きます。社内に見せたり一般公開するには、
  別途サーバー（AWS / GCP など）にAppwriteを置く作業が必要です。これは
  「サンプルで手応えを得て、本格的に進めると決めた後」で構いません。
- ここで作った「データ構造（DATA_MODEL.md）」と「画面」は、将来の土台になります。
  仮に開発者がAppwriteではなく別の仕組み（Supabase等）を選んでも、
  作るものの中身としてそのまま引き継げます。無駄になりません。
